## Objetivo Geral

* O objetivo geral do projeto "Just do Eat" é desenvolver uma plataforma web intuitiva de delivery que conecta usuários a restaurantes e, de forma simultânea, auxilia instituições carentes. 
* A solução busca unir a praticidade e a conveniência do pedido de refeições online com a responsabilidade social, gerando doações automaticamente através das compras realizadas no sistema por meio de um programa chamado "Parcela Amiga".
<br>

## Objetivos Específicos

* Simplificar o processo de compra e entrega de produtos alimentícios, promovendo acessibilidade digital para diferentes perfis de usuários.
* Ampliar o alcance de ações solidárias no cotidiano das pessoas, resolvendo a dificuldade que instituições carentes enfrentam para obter alimentos fora de datas comemorativas.
* Destinar, através do mecanismo "Parcela Amiga", uma porcentagem dos lucros de cada transação para um fundo solidário que é revertido em alimentos e suprimentos básicos para as instituições parceiras cadastradas.
* Contribuir ativamente para os Objetivos de Desenvolvimento Sustentável (ODS) da ONU, com foco no ODS 2 (Fome Zero e Agricultura Sustentável), ODS 3 (Boa Saúde e Bem-Estar) e ODS 9 (Indústria, inovação e infraestrutura).
* Fornecer um ecossistema completo que inclui a gestão de perfis (clientes, restaurantes e administradores), sistema de carrinho e checkout, rastreamento de entregas em tempo real e um sistema de avaliação de serviços.
<br>

## Descrição das Interfaces já desenvolvidas

### Homepage (Layout Web):

* A tela inicial conta com uma interface amigável que apresenta uma barra de pesquisa, divisão rápida por categorias (como Lanches, Marmitas e Bebidas), uma vitrine destacando os melhores restaurantes e uma área de chamada (Call to Action) para engajar o usuário a contribuir com a causa social.

### Suporte / FAQ:

* Uma página focada no atendimento ao usuário, contendo uma barra de pesquisa de ajuda, menus expansíveis para dúvidas frequentes (como etapas para ser parceiro ou cadastrar estabelecimento) e um espaço para envio de feedback para a melhoria do software.

### Quem Somos Nós:

* Uma tela institucional que exibe claramente a missão do projeto e detalha visualmente os Objetivos de Desenvolvimento Sustentável (ODS) da ONU que a plataforma atende.

### Login e Registro (Cadastro):

* Telas limpas para autenticação de usuários, A interface de registro é abrangente e se adapta ao tipo de usuário (Cliente ou Restaurante), capturando dados essenciais como Nome/Razão Social, CPF/CNPJ, Endereço completo, Telefone e Horários de Abertura/Fechamento.

### Dashboard Administrativo:

* Um painel de controle focado na gestão do negócio (voltado para restaurantes/administração).
* A aba "Visão Geral" exibe métricas vitais como número de pedidos ativos, valor das vendas do dia, ranking de itens mais vendidos e histórico de avaliações recentes.

### Listar (Cadastro / Edição de Produtos):

* Uma interface dupla que funciona como gestor de cardápio. Do lado esquerdo, lista os produtos já disponíveis com foto, nome, descrição, preço e status (ativo).
* Do lado direito, apresenta um formulário para cadastrar novos produtos ou editar os existentes, definindo categorias e opções adicionais.
